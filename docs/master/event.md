# イベント(パークマスター向け) {#title}

この章では、シナリオ内で発生する、各種イベントにおいて記載するべき事項について書き記す。

今作に限らず、TRPG内でのイベントにおいて、重要な要素は「条件」と「演出」、そして「処理」である。
むろん今作でもそれに変わりはないが、この際書き記しておくとPMを混乱させることなくイベントを伝えられるであろう。

## 「条件」 {#branch}

条件、またはフラグとは、そのイベントを発生させるために必要な、アイテム、発動済みイベント、研究員の選択などといった条件である。
イベント作成の際に必ずなければいけない要素ではないが、条件分岐があるときは、混乱しないよう記載しておこう。

フラグに必要になるであろう要素の例を以下に記載する。

### 研究員の選択 {#choose_of_researcher}

研究員が選択した行動、言動によって、その後のイベントの流れを変える。
最も一般的な条件分岐の方法。

#### 分岐例 {#rchoose_of_researcher_example}

- 敵かどうかわからないNPCに質問などで揺さぶりをかける。
- 遭難したNPCを助ける際に、声をかけて不安を紛らわせる。
- うさんくさいNPCにアイテムを引き渡す際、実際に渡すか否かを選ばせる。

### 研究員のパラメータ {#parameter_of_researcher}

研究員がイベント内である事象に行きついた時、研究員のパラメータによって追加で情報を渡す。
これを利用することで、「専門家以外にはわからないヤバい状態」、「特定の研究員以外にはわからない特殊なヒント」などの分岐を行うことができる。
知力判定を利用するとそれっぽくなるだろう。

#### 分岐例 {#parameter_of_researcher_example}

「自然」判定：
:   - NPCから取得を頼まれていた薬草を識別する。
    - 危険な毒物を識別する。

「歴史」判定：
:   - ジャパリパーク内で起きた事件を元に、敵の弱点を推察する。
    - パークの外で起きた事件と、現在のセッションで調査中の事件の関係を導く。

### 特殊なアイテム {#special_item}

研究員または研究員チームが事前に取得しているアイテムによって、条件分岐を発生させる。
アイテムの他にも、お金などを利用しても問題はない。

#### 分岐例 {#special_item_example}

- 薬草やアニマルラムネを事前に取得し、元気のないアニマルガールにプレゼントする。
- どこかで拾った特殊な硬貨を利用して、別の特殊アイテムなどと交換する。

### 発動済みイベント {#completed_event}

事前に発生したイベントまたはその結果によって、次のイベントの結果も分岐させる。

#### 分岐例 {#completed_event_example}

- 依頼達成に対して、忘れた頃に時間差でご褒美を与える。
- 事前に出したミッションの成果に応じて、難易度を事前に調整する。

## 「演出」 {#direction}

演出とは、イベント時に発生する五感上の情報や、そこで発生する会話などを表す。

といっても、当然ながらほとんどの場合ではそれらの情報が本当に発生するわけではなく、テキスト文で表される。
ここに関してはある程度自由に記載することができるが、ステータス、パラメータ操作などと混同させないためにも、後述の「処理」とは分離して記載した方がよいだろう。

## 「処理」 {#process}

処理とは、イベント時に発生する、ゲーム進行に影響する事象を表す。
具体的には、以下のようなものが当てはまる。

- 研究員、NPCのステータス・パラメータなどの操作
- アイテムの取得、剥奪
- イベント中限定の買物
- 特殊ルールの追加

### ステータス操作 {#status_control}

その名の通り、研究員またはNPCに対して、強制的にバフ及びデバフをかける。

#### 使用場面 {#status_control_example}

- 研究員の明確なピンチの演出
    - ぬかるみに足をとられて、弱体化した状態で戦闘に挑まなければいけない
    - 倒したと思った敵がパワーアップして復活した
- 敵の弱体化演出
    - 急所を突かれて悶絶し、攻撃力が下がった
    - 事前に用意しておいた毒が徐々に効いてきて、攻撃が通るようになった
- 事実上の時間制限追加
    - 救助対象の病状が悪化し、少しずつ衰弱していく

### アイテム取得 {#add_item}

アイテムを強制的に与え、または奪う。
重要アイテムだけではなく、消耗品などの一般的なアイテムの取得・剥奪でも問題ない。

#### 使用場面 {#add_item_example}

- わかりやすいフラグ管理
    - カギを与えることで、開かなかった扉を開けられるようになる
    - 次のイベントで、与えたアイテムを所持していた時に、それを利用して条件分岐を行う
- 簡単なお使い
    - 病気のNPCのために薬草を購入する
    - 絶対に壊してはいけないアイテムを持たせる

### イベント中限定の買物 {#purchase}

イベント中に、買物を挟む。
ラインナップは事前に設定する必要があるが、重要アイテムを設定させることもできる。

#### 使用場面 {#purchase_example}

- ほかのイベントに必要なアイテムを売買する
- この先のボス戦と相性のいいアイテムを購入させるよう促す
- いつもより安い値段で、弱い武器を売りさばく
- ゴールド以外の手段で取引を行う
    - 物々交換
    - 専用通貨
    - サンドスター提供
    - 代償付き人間度回復
